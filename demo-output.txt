FINAL: Here is the synthesized common advice agreed upon across all three sources on Python asyncio best practices:

1. **Use `asyncio.run()` as the main entry point.** All sources agree that `asyncio.run()` is the correct, idiomatic way to launch an asyncio program, rather than manually managing the event loop.

2. **Never block the event loop.** A consistent warning across all sources: avoid calling blocking (synchronous) code — such as long-running CPU-bound operations, `time.sleep()`, or blocking I/O — inside async functions, as it stalls all concurrency.

3. **Use `async`/`await` consistently throughout your code.** All sources emphasize that coroutines must be properly defined with `async def` and called with `await` to participate correctly in the event loop; mixing sync and async carelessly is a top pitfall.

4. **Run concurrent tasks with `asyncio.gather()` or `asyncio.create_task()`.** Rather than awaiting coroutines sequentially, all sources recommend using these constructs to run multiple tasks concurrently and realize the actual performance benefits of asyncio.

5. **Avoid creating unnecessary tasks or "fire and forget" without tracking them.** All sources caution against spawning tasks without holding a reference, as untracked tasks can silently fail and exceptions may be lost.

6. **Handle cancellation and shutdown gracefully.** All three sources highlight the importance of properly handling `asyncio.CancelledError` and setting up signal handlers for clean shutdown, ensuring resources are released and tasks are properly cancelled on exit.

7. **Use asyncio-compatible libraries for I/O.** All sources stress that the benefits of asyncio are only realized when using async-native libraries (e.g., `aiohttp`, `asyncpg`) — using blocking libraries for network or database I/O defeats the purpose entirely.
